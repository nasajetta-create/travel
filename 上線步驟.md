# 旅遊分帳 上線步驟（T0710-08）

上線包內容：`index.html`（App 本體）、`sw-tsplit.js`（離線快取）、`manifest.webmanifest`（PWA 設定）、`icon-512 / 192 / 180.png`（App 圖示：地球儀航線款）。
**七個檔案全部**放進 repo 根目錄。

## 一、GitHub Pages 部署（跟 G95 同套路，全程 GitHub Desktop）

1. GitHub 網頁 → New repository → 名稱 `travel`（Public）→ Create。
2. GitHub Desktop → Clone repository → 選 `nasajetta-create/travel`。
3. 用檔案總管把 `travel上線包` 裡的**全部檔案**複製進 clone 下來的 `travel` 資料夾。
4. GitHub Desktop：Summary 填 `T0710-08 首次上線` → Commit → Push origin。
5. GitHub 網頁 → travel repo → Settings → Pages → Branch 選 `main`、資料夾 `/ (root)` → Save。
6. 等 1～2 分鐘，網址：`https://nasajetta-create.github.io/travel/`
   （若 Actions 顯示 Queued/失敗，推一個新 commit 重觸發，同 G95 經驗。）

## 二、Firebase 雲端同步（免費 Spark，一次設定）

沒做這步 App 也能用（單機模式），但旅伴無法用代碼加入。

1. https://console.firebase.google.com → 建立專案（名稱隨意，例 `travel-split`）→ 不用開 Analytics → **方案保持 Spark 免費**。
2. 專案總覽 → `</>`（新增網頁應用程式）→ 註冊 → 複製畫面上的 `firebaseConfig = {...}` 整段。
3. 打開 repo 的 `index.html`，搜尋 `window.FIREBASE_CONFIG = null;`，把 `null` 換成貼上的 `{...}` 物件（檔內註解處也有說明）。
4. Firebase 左側 建構 → Authentication → 開始使用 → Sign-in method → 啟用「匿名」。
5. 左側 建構 → Firestore Database → 建立資料庫（正式版模式、地區選 asia-east1）→ 規則改成：
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /trips/{code} {
         allow read, write: if request.auth != null;
       }
     }
   }
   ```
   → 發布。
6. GitHub Desktop 再 Commit＋Push 一次（Summary：`啟用 Firebase`）。
7. 開網址確認頂欄變「☁️ 同步中」。

## 三、手機安裝與旅伴加入

- 手機瀏覽器開網址 → 分享/選單 →「加入主畫面」→ 像 App 一樣全螢幕使用、離線可記帳。
- 你建立旅程 → 旅程頁複製 6 碼代碼給旅伴 → 旅伴開同網址 → 旅程 → 輸入代碼加入 → 各自在「我是誰」點自己的徽章。

## 四、費用與安全

- GitHub Pages 免費；Firebase Spark 免費（這種用量遠碰不到上限），**不會產生月費**（與 g95-work 專案完全分開）。
- 知道旅程代碼的人才看得到該旅程；不想被亂入就別公開代碼。
- 私人帳只存各自手機，換手機前記得「旅程頁 → 匯出備份」。
